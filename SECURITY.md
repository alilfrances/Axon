# Security Policy

## Supported Versions

Security fixes are provided for the latest released version of Axon.

## Reporting a Vulnerability

Do not open a public issue for suspected vulnerabilities.

Use GitHub private vulnerability reporting:

<https://github.com/AlilKuizon/Axon/security/advisories/new>

If private reporting is not available, contact the maintainer through the GitHub profile associated with this repository and include only enough detail to establish contact. Share exploit details after a private channel is established.

## Scope

Security-sensitive areas include:

- MCP server tool inputs and outputs
- process execution and sandbox helpers
- patch application and verification workflows
- SAST, triage, and refutation logic
- dependency and plugin installation paths

## Response Targets

- Acknowledge: within 7 days
- Initial assessment: within 14 days
- Fix or mitigation target for confirmed high severity issues: within 30 days

## Disclosure

Confirmed vulnerabilities should be disclosed through a GitHub Security Advisory after a fix or mitigation is available. Public discussion before coordinated disclosure may put users at risk.
